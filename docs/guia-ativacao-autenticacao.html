Spend Analysis v3 — Liberação de acesso via grupo de segurança

Para: Diretor(a) / Administrador Azure  |  Tempo: ~20 min

1. Registrar o aplicativo

  1. Acesse entra.microsoft.com
  2. Aplicativos → Registros de aplicativo → + Novo registro
  3. Preencha:
    NomeSpend Analysis v3
    Tipos de contaContas somente neste diretório organizacional
    PlataformaSPA (Aplicativo de Página Única)
    URI de redirecionamentohttps://salmon-beach-05662180f.6.azurestaticapps.net
  4. Clique Registrar
  5. Vá em Permissões de API → confirme que User.Read está listada → clique Conceder consentimento de administrador
Anote na página "Visão geral" do app: ID do aplicativo (cliente) e ID do diretório (locatário).

2. Criar o grupo de segurança

  1. Grupos → + Novo grupo
  2. Preencha:
    TipoSegurança
    NomeSpendAnalysis-Usuarios
    AssociaçãoAtribuído
  3. Em Membros, adicione todos os consultores e admins que devem ter acesso
  4. Clique Criar
Anote o ID do objeto do grupo (na página "Visão geral" do grupo).

3. Restringir acesso ao grupo

Passo mais importante. Sem ele, qualquer pessoa do tenant consegue logar.
  1. Aplicativos → Aplicativos empresariais → abra Spend Analysis v3
  2. Em Propriedades: mude "Atribuição de usuário obrigatória?" para Sim → Salvar
  3. Em Usuários e grupos → + Adicionar → selecione SpendAnalysis-Usuarios → Atribuir

4. Incluir grupos no token

  1. Volte em Registros de aplicativo → Spend Analysis v3 → Manifesto
  2. Localize "groupMembershipClaims" e altere de null para "SecurityGroup"
  3. Salvar

5. Enviar para a equipe técnica

NomeValor
TENANT_ID (ID do diretório) 
CLIENT_ID (ID do aplicativo) 
GROUP_ID (ID do objeto do grupo) 

Gestão futura de acesso

Adicionar usuário: Grupos → SpendAnalysis-Usuarios → Membros → + Adicionar

Remover usuário: Grupos → SpendAnalysis-Usuarios → Membros → selecione → Remover
