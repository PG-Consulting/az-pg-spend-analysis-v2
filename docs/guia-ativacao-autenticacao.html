Configuração do Azure Entra ID

Spend Analysis v3 — Ativação do login corporativo

Para: Administrador Azure  |  Tempo estimado: ~20 min

1. Registrar o aplicativo

  1. Acesse entra.microsoft.com
  2. No menu lateral: Identidade → Aplicativos → Registros de aplicativo → + Novo registro
  3. Preencha:
    NomeSpend Analysis v3
    Tipos de conta com suporteContas somente neste diretório organizacional
    URI de redirecionamento — PlataformaSelecione Aplicativo de página única
    URI de redirecionamento — URLhttps://salmon-beach-05662180f.6.azurestaticapps.net
  4. Clique Registrar
  5. Na tela seguinte ("Visão geral"), vá em Gerenciar → Autenticação
  6. Na seção "Aplicativo de página única", clique Adicionar URI e adicione: http://localhost:3000
  7. Clique Salvar
Anote dois valores da página "Visão geral": ID do aplicativo (cliente) e ID do diretório (locatário).

2. Expor a API do aplicativo

Obrigatório. Sem este passo o login não funciona.
  1. Ainda no app, em Gerenciar → Expor uma API
  2. Clique Adicionar ao lado de "URI de ID do Aplicativo" — aceite o valor sugerido e clique Salvar
  3. Clique + Adicionar um escopo e preencha:
    Nome do escopoaccess_as_user
    Quem pode consentirAdministradores e usuários
    Nomes de exibição e descrições (4 campos)Preencha todos com: Acessar Spend Analysis
    EstadoHabilitado
  4. Clique Adicionar escopo
  5. Em Gerenciar → Permissões de API → confirme que User.Read aparece na lista → clique Conceder consentimento do administrador → Sim

3. Criar grupo de segurança

  1. No menu lateral: Identidade → Grupos → Todos os grupos → + Novo grupo
  2. Preencha:
    Tipo de grupoSegurança
    Nome do grupoSpendAnalysis-Usuarios
    Tipo de associaçãoAtribuído
  3. Clique em Membros → + Adicionar → adicione as pessoas que devem ter acesso
  4. Clique Criar
Anote o ID do objeto do grupo (página "Visão geral" do grupo).

4. Restringir acesso ao grupo

Sem este passo qualquer pessoa da organização consegue entrar no sistema.
  1. No menu lateral: Identidade → Aplicativos → Aplicativos empresariais → Todos os aplicativos → abra Spend Analysis v3
  2. Propriedades → mude "Atribuição necessária?" para Sim → Salvar
  3. Usuários e grupos → + Adicionar usuário/grupo → selecione o grupo SpendAnalysis-Usuarios → Atribuir

5. Incluir grupos no token

  1. Volte em Identidade → Aplicativos → Registros de aplicativo → abra Spend Analysis v3
  2. Gerenciar → Configuração de token → + Adicionar declaração de grupos
  3. Marque Grupos de segurança → Adicionar

6. Enviar para a equipe técnica

Preencha e envie os três valores abaixo:

DadoOnde encontrarValor
TENANT_IDVisão geral do app → "ID do diretório" 
CLIENT_IDVisão geral do app → "ID do aplicativo" 
GROUP_IDVisão geral do grupo → "ID do objeto" 

Gestão futura de acesso

Adicionar usuário: Grupos → SpendAnalysis-Usuarios → Membros → + Adicionar

Remover usuário: Grupos → SpendAnalysis-Usuarios → Membros → selecione → Remover
